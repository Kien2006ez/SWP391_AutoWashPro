\chapter{Thiết Kế Kiến Trúc Và Hệ Thống}

Nhóm tiến hành phân tích hệ thống dữ liệu cốt lõi, kiến trúc mã nguồn tổng quan và giao diện người dùng dựa trên các yêu cầu chức năng của hệ thống Quản lý Rửa xe Thông minh (\textbf{AutoWashPro}). Toàn bộ quy trình thiết kế được bám sát theo tiến độ phân rã các thẻ công việc trên bảng quản trị Jira.

% =================================================================
\section{Thiết kế Kiến trúc hệ thống (System Architecture Diagram)}
Để đảm bảo hệ thống có khả năng mở rộng, bảo mật cao và vận hành ổn định, nhóm triển khai hệ thống theo mô hình kiến trúc 3 lớp tiêu chuẩn (3-Tier Architecture) kết hợp với các thành phần xử lý tác vụ tự động chạy ngầm.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=0.85\textwidth]{system_architecture.png}
    \caption{Sơ đồ Kiến trúc hệ thống 3 lớp (3-Tier Architecture) - AutoWashPro}
    \label{fig:system_architecture}
\end{figure}

Sơ đồ kiến trúc mô tả rõ ràng sự tách biệt giữa tầng hiển thị Client (giao diện Web HTML5/CSS3/JavaScript thuần), tầng xử lý logic nghiệp vụ trung tâm Server API (\textbf{Python/FastAPI}, chạy trên ASGI server Uvicorn) và tầng lưu trữ dữ liệu bền vững Database (\textbf{MySQL}). Đồng thời, hệ thống tích hợp bộ lập lịch Cron Job (APScheduler) chạy ngầm để thực thi các tác vụ định kỳ tự động (quét hạng thành viên) và phân tách riêng phân hệ log hành vi phục vụ nghiên cứu khoa học.

\section{Thiết kế Cơ sở dữ liệu và Sơ đồ ERD}
Hệ thống sử dụng cơ sở dữ liệu quan hệ MySQL nhằm đảm bảo tính toàn vẹn dữ liệu, tính nhất quán trong các luồng nghiệp vụ phức tạp như xác thực tài khoản, đặt lịch hẹn và xử lý điểm thưởng.

\subsection{Sơ đồ mối quan hệ thực thể (ERD Diagram)}
Sơ đồ ERD dưới đây biểu diễn mối quan hệ logic, kiểu dữ liệu, các ràng buộc khóa chính (PK), khóa ngoại (FK) giữa các bảng thực thể đã được chuẩn hóa theo đúng mô hình đặc tả hệ thống.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=0.95\textwidth]{erd_diagram.png}
    \caption{Sơ đồ mối quan hệ thực thể ERD - Phân hệ cốt lõi AutoWashPro}
    \label{fig:erd_diagram}
\end{figure}

\subsection{Công nghệ ORM và Quản lý dữ liệu}

Hệ thống sử dụng \textbf{SQLAlchemy} (phiên bản 2.x) làm công cụ ORM (Object Relational Mapping) kết hợp \textbf{Alembic} để quản lý migration schema. SQLAlchemy cho phép ánh xạ trực tiếp các bảng dữ liệu MySQL thành các lớp Python (\textit{Declarative Model}) khai báo tại \texttt{app/models.py}, giúp giảm thiểu việc thao tác SQL thủ công, tăng khả năng bảo trì mã nguồn và đảm bảo tính nhất quán dữ liệu.

Tầng \textit{Service} (\texttt{app/services/}) chịu trách nhiệm xử lý các quy tắc nghiệp vụ, truy vấn dữ liệu thông qua \texttt{Session} của SQLAlchemy (được inject qua dependency \texttt{get\_db()} khai báo tại \texttt{app/database.py}), trước khi trả kết quả về tầng \textit{Router} (\texttt{app/routers/}) -- tương đương tầng Controller trong các framework khác.

\section{Thiết kế sơ đồ tuần tự chi tiết (Sequence Diagrams)}
Sơ đồ tuần tự mô tả chi tiết dòng chảy tương tác theo trục thời gian giữa Client, Server API, Hệ thống Cơ sở dữ liệu và Tác vụ chạy ngầm đối với 4 quy trình nghiệp vụ cốt lõi của hệ thống.

\subsection{Luồng nghiệp vụ Đặt lịch rửa xe (Book Wash Sequence)}
Mô tả trình tự tương tác khi Khách hàng chọn thông tin xe, dịch vụ, ngày giờ. Hệ thống Backend thực hiện kiểm tra phân hạng thành viên để áp quy tắc cửa sổ thời gian (\textit{Booking Window}) và kiểm tra trạng thái trống của slot trước khi lưu lịch hẹn.

\section{Thiết kế API REST Specification}
Hệ thống sử dụng kiến trúc RESTful API phía Backend (FastAPI) tương tác với Frontend thông qua các endpoint HTTP chuẩn (không sử dụng tiền tố \texttt{/api}). Dưới đây là đặc tả các API endpoints, phân biệt rõ giữa phần \textbf{đã triển khai} và phần \textbf{đang trong kế hoạch} theo tiến độ Jira.

\subsection{API Authentication Endpoints (Feature 1 -- Đã triển khai)}

\subsubsection{POST /auth/register - Đăng ký tài khoản mới}
\begin{verbatim}
Request Body:
{
  "username": "user01",
  "password": "SecurePass123"
}

Response (200 OK):
{
  "message": "Register success"
}

Error Response (400 Bad Request):
{
  "detail": "Username already exists"
}
\end{verbatim}

\subsubsection{POST /auth/login - Đăng nhập hệ thống}
\begin{verbatim}
Request Body:
{
  "username": "user01",
  "password": "SecurePass123"
}

Response (200 OK):
{
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "username": "user01",
  "role": "Customer"
}

Error Response (401 Unauthorized):
{
  "detail": "Invalid username or password"
}
\end{verbatim}

\subsubsection{GET /bookings/\{id\}, PUT /bookings/\{id\}, DELETE /bookings/\{id\}}
Lần lượt tra cứu chi tiết, cập nhật và hủy lịch hẹn theo \texttt{id}. Trả về lỗi \texttt{404 Not Found} nếu lịch hẹn không tồn tại.

\subsection{API Loyalty, Vehicle, Promotion \& Admin Endpoints (Feature 2, 4, 5, 6 -- Kế hoạch triển khai)}
Các endpoint dưới đây thuộc phạm vi FR đã đặc tả tại Chương 3 nhưng \textbf{chưa được lập trình} tại thời điểm biên soạn báo cáo (các router tương ứng trong mã nguồn hiện đang trống):

\begin{itemize}
    \item \texttt{GET/POST/PUT/DELETE /vehicles} -- Quản lý hồ sơ xe (M5).
    \item \texttt{GET /loyalty/points}, \texttt{POST /loyalty/redeem} -- Xem số dư và đổi điểm thưởng (M4).
    \item \texttt{GET/POST/PUT/DELETE /promotions} -- Quản lý khuyến mãi (M5).
    \item \texttt{GET /admin/dashboard}, \texttt{GET/PUT /admin/bookings}, \texttt{GET/PUT /admin/users} -- Quản trị trung tâm (M3).
    \item \texttt{POST /survey}, \texttt{GET /survey/export} -- Khảo sát và xuất dữ liệu nghiên cứu (M6, M7).
\end{itemize}
