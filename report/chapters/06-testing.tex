\chapter{Kiểm Thử Hệ Thống AutoWashPro}

\section{Kế hoạch & Chiến lược kiểm thử}
Quá trình kiểm thử hệ thống AutoWashPro được thực hiện theo các cấp độ: Unit Tests (kiểm thử từng module), Integration Tests (kiểm thử flow liên kết), System Tests (kiểm thử toàn bộ hệ thống) và User Acceptance Tests (kiểm thử với người dùng thực). Tất cả các test case được ghi nhận trên Jira board và được tự động hóa thông qua CI/CD pipeline.

\section{Integration Tests}

\subsection{End-to-End Flow Tests}

\begin{enumerate}
    \item \textbf{TC-INT-01: Full Register \& Login Flow}
    \begin{itemize}
        \item Register với email mới → Verify OTP → Login → Nhận JWT token
        \item Verify: Token valid, user info returned, can access dashboard
    \end{itemize}
    
    \item \textbf{TC-INT-02: Complete Booking Flow}
    \begin{itemize}
        \item Add vehicle → Select service → Pick date/time → Confirm booking → Auto create invoice
        \item Verify: Booking status=pending, Invoice created, email sent
    \end{itemize}
    
    \item \textbf{TC-INT-03: Payment Processing Flow}
    \begin{itemize}
        \item Initiate payment → Redirect to VNPay → Receive callback → Update invoice
        \item Verify: Invoice status=paid, email confirmation, refund available
    \end{itemize}
    
    \item \textbf{TC-INT-04: Cancel Booking with Refund}
    \begin{itemize}
        \item Complete booking → Make payment → Cancel within 6 hours → Process refund
        \item Verify: Booking cancelled, refund initiated, payment status=refunded
    \end{itemize}
\end{enumerate}

\section{Security Tests}

\subsection{Security Validation Tests}

\begin{enumerate}
    \item \textbf{TC-SEC-01: Password Hashing}
    \begin{itemize}
        \item Register user → Check database for password\_hash
        \item Verify: Password hashed with bcrypt (not plaintext), cost \(\geq\) 12
    \end{itemize}
    
    \item \textbf{TC-SEC-02: JWT Token Security}
    \begin{itemize}
        \item Login → Extract JWT token → Decode (should fail without secret)
        \item Verify: Token contains user\_id, role; signed with HS256
    \end{itemize}
    
    \item \textbf{TC-SEC-03: SQL Injection Prevention}
    \begin{itemize}
        \item Input: "'; DROP TABLE users; --" in email field
        \item Verify: Rejected or escaped, no SQL error in response
    \end{itemize}
    
    \item \textbf{TC-SEC-04: XSS Prevention}
    \begin{itemize}
        \item Input: "<script>alert('XSS')</script>" in vehicle brand
        \item Verify: Escaped/sanitized, no script execution
    \end{itemize}
    
    \item \textbf{TC-SEC-05: CSRF Protection}
    \begin{itemize}
        \item POST request without CSRF token
        \item Verify: Request rejected with 403 Forbidden
    \end{itemize}
    
    \item \textbf{TC-SEC-06: Payment Signature Verification}
    \begin{itemize}
        \item Receive callback with tampered amount
        \item Verify: Signature mismatch detected, transaction rejected
    \end{itemize}
\end{enumerate}
